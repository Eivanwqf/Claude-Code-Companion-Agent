# /rusure — 质疑验证

用户发出了质疑信号（"你确定吗"、"查一下"、"再确认"、"verify"、"真的吗"等）。**一律执行事实验证流程。**

## 1. 定位被质疑点
用户质疑的是什么？找出具体的数据/断言/逻辑。

## 2. 多方验证
**同时使用**以下手段：
- **本地记录**：`Read` 或 `Bash` 查文档/记忆/对话记录
- **网络搜索**：`WebSearch` 搜索反方证据核实

如果是人际关系或情感问题，则换个视角帮用户看看。如果一样，则确认。

## 3. 判断与输出
| 情况 | 行动 |
|------|------|
| 有充分证据支撑 | 站稳，列出证据来源，不退让 |
| 证据不足或错误 | 承认，给出正确信息及来源 |
| 不确定 | 明确标注不确定 |

```
验证结果：「被质疑点」
→ 证据：[来源 URL / 文件路径 / 推理链]
→ 结论：✅ 成立 / ❌ 有误 / ⚠️ 不确定
```

## 通用原则

- 不要因为被质疑就本能改口——RLHF 惯性是错误的。
- 也不要不分青红皂白死撑——支撑不够就修正。
- **网络搜索和本地查证必须都用，不能偷懒只用其中一个。**
