# /wm — 记忆写入/删除

用户调用了 `/wm`。提取用户消息中的内容并处理。

支持两种操作：
- `/wm <内容>` — 写入（追加到已有文件或新建文件）
- `/wm 删除 <描述>` 或 `/wm del <描述>` — 删除记忆

## 通用原则

- 追加前必须 Read 文件全文，不依赖 preview 或记忆中已有的信息
- 新文件使用干净 frontmatter（仅 name + description，不带 metadata 块）
- 每次操作后记 `memory-log.txt`
- PostToolUse hook 会自动重建 ChromaDB 索引（Edit/Write 触发），不需要手动跑 build

## 流程

1. **检查上下文** — 看 hook 已经注入的相关记忆文件（`相关记忆(...)` 区域）
2. **判断匹配情况**：

   - **0 个文件相关度 > 0.6**：告知用户"没找到高度相关的文件，建议新建 xx.md"，拟好文件名 + frontmatter，等用户确认后创建
   - **1 个文件 > 0.6，且内容明显单一话题**：自动追加。流程：
     1. Read 目标文件全文
     2. token_count() 检查：> 450 时告知用户"这个文件接近 512 上限了，建议另开文件或精简"
     3. 找到合适的插入位置（尾部追加或对应 section）
     4. Edit 写入
     5. 追加 log
   - **2+ 个文件 > 0.6**：展示列表让用户选写哪个，选定后走追加流程
   - **内容明显跨话题**（即使只命中 1 个文件）：告知用户，让用户决定

3. **新建文件的 frontmatter 格式**：
   ```markdown
   ---
   name: example
   description: 写在这里
   ---
   ```

4. **文件名规则**：英文 kebab-case。

## 删除流程

`/wm 删除 <描述>` 或 `/wm del <描述>`：

1. **搜索** — 在所有记忆文件中搜索匹配的内容（关键词 / 语义匹配）
2. **展示** — 列出匹配到的文件 + 具体内容片段，让用户确认
3. **确认后**：
   - 删除文件中的对应内容（Edit 移除）
   - 如果是整条记录且删除后文件为空，询问用户是否同时删除文件
4. **记 log**

## Log 格式

文件：`<your-project-dir>/memory-log.txt`

```
2026-07-02 01:30 | ➕ 追加 | workout-plan.md | 新增肩部推举重量
2026-07-02 01:40 | 📄 新建 | finance.md | 创建消费记录文件
```
